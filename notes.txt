  Scaling is nearly perfect up to 4 threads, good at 8, then falls off a cliff at 20. There are several possible causes, and they require different
  diagnostic approaches:

  1. Work imbalance (likely the biggest factor for Mandelbrot)

  Mandelbrot is pathologically uneven — pixels inside the set run 256 iterations while exterior pixels bail out after a handful. Tiles near the boundary
  are 10-100x more expensive than tiles in the black interior or far exterior. With 900/16 = ~56 tiles across 20 threads, a few unlucky threads get stuck
   on expensive boundary tiles while others finish early and idle. You can test this directly by benchmarking the gradient kernel (perfectly uniform
  work) at the same thread counts — if scaling improves dramatically, work imbalance is the culprit.

  2. Memory bandwidth saturation

  At 20 threads all writing to a 4.3MB framebuffer, you may be hitting memory bandwidth limits. perf stat can show L2/L3 cache miss rates and memory
  bandwidth utilization.

  3. Hardware topology

  Your machine has 20 cores on AArch64. If there's a big.LITTLE configuration (performance + efficiency cores) or NUMA boundaries, threads on slower
  cores or remote memory drag down the average. lscpu would clarify this.

  4. Rayon coordination overhead

  Work-stealing has a cost. With 56 tiles and 20 threads, each thread only gets ~3 tiles — the ratio of useful work to scheduling overhead shrinks.

  Here's how I'd approach the investigation:

  Quick experiments (no code changes):
  - Bench the gradient kernel at 1/4/8/20 threads — isolates work imbalance
  - lscpu to check core topology
  - perf stat on a bench run to see IPC, cache misses, branch mispredictions

  Instrumentation (small code changes):
  - Per-tile timing: record how long each tile takes, report min/max/stddev to quantify imbalance
  - Configurable tile size: try TILE_HEIGHT=4 vs 16 vs 64 to see how granularity affects scaling
  - A "synthetic" uniform-cost kernel that does a fixed number of iterations per pixel regardless of position
